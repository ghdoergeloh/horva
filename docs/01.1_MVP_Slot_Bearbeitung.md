# Increment 1.1: Slot-Bearbeitung (Ergänzung zu MVP)

**Ziel:** Slots können über die CLI eingesehen und ihre Start-/Endzeitpunkte bearbeitet werden. Bei Auswirkung auf Nachbar-Slots wird ein Anpassungsvorschlag angezeigt.

**Vorbedingungen:** Increment 1 (MVP) abgeschlossen.

**Bezug:** Setzt die Anforderungen F-SL-06 (Zeiten bearbeiten) und F-SL-07 (Nachbar-Slot-Warnung) um.

---

## 1. CLI-Erweiterung: Slot-Befehle

### 1.1 Befehlsübersicht

```
Slots:
  tt slot list [Zeitraum]       Slots auflisten                        (Alias: tt sl ls)
  tt slot edit <ID>             Slot bearbeiten                        (Alias: tt sl edit)
  tt slot delete <ID>           Slot löschen                           (Alias: tt sl rm)
  tt slot split <ID> <Zeit>     Slot an Zeitpunkt aufteilen            (Alias: tt sl split)
```

---

### 1.2 tt slot list – Slots auflisten

```
tt slot list [Zeitraum] [Optionen]

Zeitraum:
  (ohne)             Heute
  today              Heute
  yesterday          Gestern
  <Datum>            Bestimmter Tag
  <Von>..<Bis>       Zeitraum

Optionen:
  -p, --project <ID>    Nur Slots eines Projekts
  --tasks-only          Nur Slots mit Task
```

**Beispiel:**

```bash
$ tt slot list
  Slots for today (Thu, 2026-02-19):

  ID   From  - To    | Time   | Task                         | Project
  281  08:00 - 08:54  | 0:54h  | -                            |
  282  08:54 - 09:30  | 0:36h  | #34 API Refactoring v2       | Backend
  283  09:30 - 10:15  | 0:45h  | #47 Bug in Login fixen       | Backend
  284  10:15 - 10:45  | 0:30h  | #28 Weekly Meeting vorber... | Orga
  285  10:45 - 11:00  | 0:15h  | -                            |
         -- break 11:00 - 11:30 (0:30h) --
  286  11:30 - 12:30  | 1:00h  | #34 API Refactoring v2       | Backend
  287  12:30 -  ▶     | 1:45h  | #34 API Refactoring v2       | Backend

  7 slots, 1 break (0:30h)
```

---

### 1.3 tt slot edit – Slot bearbeiten

```
tt slot edit <ID> [Optionen]

Optionen:
  -s, --start <Zeit>       Startzeitpunkt ändern (z. B. "08:30", "08:30 2026-02-19")
  -e, --end <Zeit>         Endzeitpunkt ändern
  -t, --task <Task-ID>     Task zuweisen oder ändern
  --no-task                Task-Zuordnung entfernen
```

**Beispiel – Einfache Bearbeitung (kein Nachbar betroffen):**

```bash
$ tt slot edit 281 -s 07:45
✎ Slot #281 updated
  Start: 08:00 → 07:45
  Duration: 0:54h → 1:09h
```

**Beispiel – Nachbar-Slot betroffen (Überlappung):**

```bash
$ tt slot edit 282 -e 09:45
⚠ Changing end of slot #282 to 09:45 would overlap with slot #283.
  Slot #283 currently starts at 09:30.

  Suggested adjustment:
    Slot #282: 08:54 - 09:45  (was 08:54 - 09:30)  #34 API Refactoring v2
    Slot #283: 09:45 - 10:15  (was 09:30 - 10:15)  #47 Bug in Login fixen

? Apply adjustment to slot #283? (y/N) y
✎ Slot #282 updated: end 09:30 → 09:45
✎ Slot #283 adjusted: start 09:30 → 09:45
```

**Beispiel – Nachbar-Slot betroffen (Lücke entsteht):**

```bash
$ tt slot edit 282 -e 09:15
⚠ Changing end of slot #282 to 09:15 would create a gap before slot #283.
  Slot #283 currently starts at 09:30.

  Options:
    [1] Keep gap (09:15 - 09:30, 0:15h break)
    [2] Adjust slot #283 start to 09:15 (no gap)

? Choose option: (1/2) 1
✎ Slot #282 updated: end 09:30 → 09:15
  New break: 09:15 - 09:30 (0:15h)
```

**Beispiel – Nachbar-Slot betroffen (Lücke wird geschlossen):**

```bash
$ tt slot edit 285 -e 11:30
⚠ Changing end of slot #285 to 11:30 would close the break before slot #286.
  Slot #286 currently starts at 11:30.

  Result:
    Slot #285: 10:45 - 11:30  (was 10:45 - 11:00)
    Slot #286: 11:30 - 12:30  (unchanged)
    Break removed: 11:00 - 11:30

? Apply? (y/N) y
✎ Slot #285 updated: end 11:00 → 11:30
  Break 11:00 - 11:30 removed.
```

**Beispiel – Task zuweisen:**

```bash
$ tt slot edit 281 -t #34
✎ Slot #281 updated
  Task: (no task) → #34 "API Refactoring v2" (Backend)
```

---

### 1.4 tt slot delete – Slot löschen

```
tt slot delete <ID>
```

**Beispiel:**

```bash
$ tt slot delete 285
⚠ This will delete slot #285 (10:45 - 11:00, no task).
  A break of 10:45 - 11:30 will result between slot #284 and #286.

? Are you sure? (y/N) y
✗ Slot #285 deleted.
```

---

### 1.5 tt slot split – Slot aufteilen

Teilt einen Slot an einem bestimmten Zeitpunkt in zwei Slots. Beide behalten den Task des Original-Slots.

```
tt slot split <ID> <Zeit>
```

**Beispiel:**

```bash
$ tt slot split 286 12:00
  Splitting slot #286 at 12:00...

  Before: 11:30 - 12:30 | 1:00h | #34 API Refactoring v2
  After:  11:30 - 12:00 | 0:30h | #34 API Refactoring v2  (Slot #286)
          12:00 - 12:30 | 0:30h | #34 API Refactoring v2  (Slot #288)

✓ Slot #286 split into #286 and #288.
```

## 2. Regeln für Nachbar-Slot-Anpassungen

Die folgenden Regeln definieren, wann eine Warnung erscheint und welche Vorschläge gemacht werden:

| Situation           | Auslöser                                                                                  | Vorschlag                                                          |
| ------------------- | ----------------------------------------------------------------------------------------- | ------------------------------------------------------------------ |
| **Überlappung**     | Endzeitpunkt von Slot A wird hinter den Start von Slot B verschoben (oder umgekehrt)      | Start von Slot B auf neues Ende von Slot A anpassen                |
| **Neue Lücke**      | Endzeitpunkt von Slot A wird vor den Start von Slot B gezogen (wo vorher keine Lücke war) | (1) Lücke als Pause akzeptieren oder (2) Start von Slot B anpassen |
| **Lücke schließen** | Endzeitpunkt von Slot A wird auf den Start von Slot B verschoben (Lücke wird geschlossen) | Änderung anzeigen, bestätigen lassen                               |
| **Kein Nachbar**    | Slot hat keinen angrenzenden Slot auf der geänderten Seite                                | Keine Warnung, direkt anpassen                                     |

Bei allen Warnungen gilt: **Keine Änderung wird ohne explizite Bestätigung des Nutzers durchgeführt.**
