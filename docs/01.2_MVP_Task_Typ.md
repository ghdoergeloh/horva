# Increment 1.2: Task-Typ – Aufgabe vs. Aktivität (Ergänzung zu MVP)

**Ziel:** Tasks können als „Aufgabe" (endlich, abarbeitbar) oder „Aktivität" (dauerhaft, wiederkehrend) angelegt werden, um sowohl konkrete Aufgaben als auch Verrechnungskonten und wiederkehrende Tätigkeiten abzubilden.

**Vorbedingungen:** Increment 1 (MVP) abgeschlossen.

---

## 1. Konzept

### 1.1 Motivation

Im Arbeitsalltag gibt es zwei Arten von Tätigkeiten, auf die Zeit gebucht wird:

- **Aufgaben:** Konkrete, abschließbare Arbeitspakete. „Bug in Login fixen", „PR #234 reviewen". Sie haben einen Anfang und ein Ende, können als erledigt markiert werden.
- **Aktivitäten:** Dauerhaft verfügbare Tätigkeitstypen oder Verrechnungskonten. „Weekly Meeting", „Wartung/Support", „Featureentwicklung", „Nachrichten checken". Sie werden nie „erledigt" und sollen immer in der Auswahl verfügbar sein.

### 1.2 Lösung: Task-Typ als Eigenschaft

Jeder Task erhält eine Eigenschaft **Typ** mit zwei Ausprägungen:

| Eigenschaft          | Aufgabe (default)                      | Aktivität                          |
| -------------------- | -------------------------------------- | ---------------------------------- |
| Lebenszyklus         | offen → erledigt → archiviert/gelöscht | aktiv → archiviert/gelöscht        |
| „Erledigt" markieren | Ja                                     | Nein (nicht möglich)               |
| In Auswahllisten     | Ausgeblendet wenn erledigt             | **Immer sichtbar** (solange aktiv) |
| Einplanen (Datum)    | Ja                                     | Nein (nicht sinnvoll)              |
| Labels               | Ja                                     | Ja                                 |
| Notizen / Links      | Ja                                     | Ja                                 |
| Projekt-Zuordnung    | Ja                                     | Ja                                 |
| Slot-Zuordnung       | Identisch                              | Identisch                          |
| Auswertung           | Identisch                              | Identisch                          |

---

## 2. Änderungen am Datenmodell

### 2.1 Task (erweitert)

Neues Feld:

- **Typ:** `aufgabe` (default) oder `aktivität`.

Geänderter Status-Lebenszyklus:

- **Aufgabe:** Offen → Erledigt → Archiviert / Gelöscht (wie bisher).
- **Aktivität:** Aktiv → Archiviert / Gelöscht. Der Status „Erledigt" existiert für Aktivitäten **nicht**.

## 3. Änderungen an der CLI

### 3.1 tt task new – Typ-Option

```
tt task new [Name] [Optionen]

Neue Option:
  --activity           Task als Aktivität anlegen (statt Aufgabe)
```

**Beispiele:**

```bash
# Aufgabe erstellen (default, wie bisher)
$ tt task new "Bug in Login fixen" -p Backend -l Bug
✚ Task #47 "Bug in Login fixen" created
  Type: task | Project: Backend | Labels: Bug

# Aktivität erstellen
$ tt task new "Weekly Meeting" -p Orga --activity
✚ Activity #48 "Weekly Meeting" created
  Type: activity | Project: Orga

$ tt task new "Wartung/Support" -p Kiteto --activity
✚ Activity #49 "Wartung/Support" created
  Type: activity | Project: Kiteto

$ tt task new "Featureentwicklung" -p Kiteto --activity
✚ Activity #50 "Featureentwicklung" created
  Type: activity | Project: Kiteto
```

### 3.2 tt task list – Typ-Filter

```
Neue Filter:
  --activities         Nur Aktivitäten anzeigen
  --tasks-only         Nur Aufgaben anzeigen (ohne Aktivitäten)
```

**Beispiel:**

```bash
$ tt task list
  Open tasks & activities:

  Activities:
  #  Name                         Project      Labels
  48 Weekly Meeting               Orga
  49 Wartung/Support              Kiteto
  50 Featureentwicklung           Kiteto

  Tasks:
  #  Name                         Project      Labels          Planned
  47 Bug in Login fixen           Backend      Bug             today
  34 API Refactoring v2           Backend
  28 Deployment vorbereiten       Backend      Release         tomorrow

  3 activities, 3 open tasks (5 more done – use --all to show)
```

### 3.3 tt start – Aktivitäten in der Auswahl

Bei der interaktiven Auswahl werden Aktivitäten in einer **eigenen Sektion** oberhalb der letzten Tasks angezeigt:

```bash
$ tt start
? What do you want to do?
  ❯ Start without task
  ── Activities ──
    #48 Weekly Meeting              (Orga)
    #49 Wartung/Support             (Kiteto)
    #50 Featureentwicklung          (Kiteto)
  ── Recent tasks ──
    #47 Bug in Login fixen          (Backend)     20min ago
    #34 API Refactoring v2          (Backend)     2h ago
    Create new task...
```

### 3.4 tt task done – Schutz für Aktivitäten

```bash
$ tt task done #48
⚠ "Weekly Meeting" is an activity and cannot be marked as done.
  Use "tt task archive #48" to hide it, or "tt task edit #48 --type task" to convert it.
```

### 3.5 tt task edit – Typ konvertieren

```
Neue Option:
  --type <aufgabe|aktivität>    Typ ändern
```

**Beispiel:**

```bash
# Aufgabe zur Aktivität machen
$ tt task edit #34 --type activity
✎ Task #34 "API Refactoring v2" converted: task → activity
  Status reset: was "open", now "active"

# Aktivität zur Aufgabe machen
$ tt task edit #48 --type task
✎ Activity #48 "Weekly Meeting" converted: activity → task
  Status: active → open
```

### 3.6 tt task plan – Schutz für Aktivitäten

```bash
$ tt task plan #48 tomorrow
⚠ "Weekly Meeting" is an activity and cannot be planned for a specific date.
  Convert it to a task first with "tt task edit #48 --type task".
```

## 4. Auswirkungen auf bestehende Features

| Feature                            | Auswirkung                                                                                                |
| ---------------------------------- | --------------------------------------------------------------------------------------------------------- |
| Slot-Zuordnung                     | Keine Änderung. Slots können Aufgaben und Aktivitäten gleichermaßen zugeordnet werden.                    |
| Auswertungen                       | Keine Änderung. Aufgaben und Aktivitäten werden identisch ausgewertet (Zeit fließt ins Projekt).          |
| Tagesübersicht (GUI)               | Aktivitäten erscheinen nicht in der Tagesplanung (da sie kein Datum haben), aber als Schnellstart-Option. |
| Kalender/GitHub/GitLab/Jira Import | Importierte Tasks sind immer vom Typ „Aufgabe".                                                           |
| Export (HR Works, Moco)            | Keine Änderung. Aktivitäten werden wie Aufgaben exportiert.                                               |
| Labels                             | Keine Änderung. Aktivitäten können Labels haben.                                                          |
| Idle Detection / Slot-Split        | Keine Änderung. Funktioniert identisch.                                                                   |
